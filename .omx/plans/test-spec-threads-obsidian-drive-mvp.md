# Test Spec: Threads → Obsidian(GDrive) 수집 서비스 (MVP)

- 문서 버전: v1.0
- 작성일: 2026-03-01
- 대상 PRD: `.omx/plans/prd-threads-obsidian-drive-mvp.md`

## 1) 테스트 목표
- 증분 수집(`last_success_at`)과 중복 skip이 정확히 동작하는지 검증
- Drive 날짜 폴더/파일명 규칙(created_at KST 기준) 준수 검증
- 부분 실패 허용 및 로그 기록 검증

## 2) 테스트 범위
### 포함
- CLI 인자 처리(`--accounts`)
- 시간 경계 처리(자정 전후)
- 마크다운 렌더링 포맷
- Google Drive 업로드 경로/파일명
- 상태 파일 갱신(`last_success_at`)

### 제외(MVP)
- 댓글 수집
- 스케줄러
- 웹 UI

## 3) 환경
- Python 3.11+
- 테스트 전용 Drive 폴더 ID
- OAuth 토큰(테스트 계정)
- 모의 Threads API 응답(정상/권한실패/레이트리밋)

## 4) 테스트 케이스

### TC-01 기본 수집 성공
- 조건: 첫 실행, 계정 2개, 게시글 3개
- 기대: 3개 파일 업로드, 날짜 폴더 생성, `last_success_at` 저장

### TC-02 중복 skip
- 조건: 같은 데이터로 연속 2회 실행
- 기대: 2회차 신규 0, skip 증가, 기존 파일 무변경

### TC-03 증분 수집
- 조건: 1회 실행 후 새 게시글 1개 추가
- 기대: 2회차에서 새 게시글만 1개 업로드

### TC-04 자정 경계
- 조건: created_at이 KST 23:59/00:01 데이터 포함
- 기대: 각기 다른 날짜 폴더에 정확히 저장

### TC-05 파일명 규칙
- 조건: author_id/post_id 고정 데이터
- 기대: `YYYY-MM-DD_{author_id}_{post_id}.md` 100% 일치

### TC-06 제목 생성 규칙
- 조건: 본문 첫 줄 120자
- 기대: 제목 80자 truncate

### TC-07 빈 본문
- 조건: 본문 empty
- 기대: 제목 `Untitled`, 파일 생성 성공

### TC-08 계정 일부 실패
- 조건: 3개 계정 중 1개 권한 오류
- 기대: 2개 계정은 정상 저장, 실패 계정 로그 기록, 프로세스 continue

### TC-09 Drive 폴더 권한 오류
- 조건: 폴더 쓰기 권한 제거
- 기대: 업로드 실패 로그 + 비정상 종료 코드

### TC-10 레이트리밋 재시도
- 조건: API 429 응답 후 성공
- 기대: 백오프 재시도 후 성공, 로그에 재시도 횟수 기록

### TC-11 CLI 계정 인자 파싱
- 조건: `--accounts a,b,c`
- 기대: 전달 계정 3개만 처리

### TC-12 상태파일 없는 첫 실행
- 조건: `collector_state.json` 없음
- 기대: 하한 시각을 실행일 KST 00:00:00으로 사용

## 5) 검증 체크리스트
- [ ] 수집 후 `last_success_at` 갱신
- [ ] 중복 파일 overwrite 없음
- [ ] 날짜 폴더 기준 = created_at(KST)
- [ ] 마크다운에 날짜/제목/본문 내용 포함
- [ ] 댓글 섹션 없음
- [ ] 실패 계정 로그 존재

## 6) 완료 기준
- 필수 케이스(TC-01~TC-08, TC-11, TC-12) 모두 PASS
- 치명 장애 없음(데이터 유실/중복 overwrite/잘못된 날짜 분류)
